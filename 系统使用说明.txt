印尼雅加达送货地图展示系统使用说明
==========================================

🗺️ 系统概述
-----------
本系统是基于云端的印尼雅加达送货地图展示平台，集成飞书多维表格数据源，
通过地图界面实时展示当日送货订单位置信息。

🔧 核心功能
-----------
1. 自动数据同步
   - 每日9点和14点自动从飞书获取送货数据
   - 支持手动刷新数据功能
   - 数据实时更新到地图显示

2. 地图可视化
   - 总部位置星形标记（固定在雅加达）
   - 订单位置红色圆形标记
   - 详细订单信息弹窗显示
   - 支持标准地图和卫星视图切换

3. 统计信息面板
   - 显示总订单数量
   - 显示总货物数量（DUS）
   - 实时数据统计

🚀 使用步骤
-----------
1. 登录系统
   用户名：One Meter
   密码：prioritaspelayanan

2. 查看订单分布
   - 左侧面板显示总订单数和货物统计
   - 地图上红色标记为当日送货订单
   - 点击标记查看详细订单信息

3. 地图操作
   - 支持地图缩放和拖拽
   - 可切换标准地图和卫星视图
   - 支持用户定位功能

4. 手动更新数据
   - 点击"🔄"按钮同步最新飞书数据
   - 系统有1分钟冷却时间防止频繁操作
   - 数据更新后地图自动刷新

📊 数据展示功能
--------------
1. 订单信息显示
   - 店主姓名
   - 联系电话
   - 货物数量（DUS）
   - 订单金额（IDR）
   - 订单类型和编号

2. 地理位置展示
   - 精确的经纬度定位
   - 清晰的地图标记
   - 详细的弹窗信息

💡 系统特性
-----------
- 🌐 云端部署：Netlify前端 + Render后端
- 📱 响应式设计：支持桌面和移动设备
- 🔄 实时同步：飞书数据自动同步到GitHub
- 💰 成本优化：免费额度内运行
- 🔒 安全访问：登录保护，会话管理
- 📍 精确定位：支持用户当前位置获取

⚠️ 注意事项
-----------
1. 环境变量要求
   - 飞书API密钥（FEISHU_APP_ID, FEISHU_APP_SECRET等）
   - GitHub访问令牌（GITHUB_TOKEN）

2. 数据格式要求
   - 飞书表格必须包含：经纬度、货物数量等字段
   - 坐标数据必须有效（非0值）
   - 日期格式为印尼雅加达时区

3. 使用限制
   - 手动更新有1分钟冷却时间
   - 仅显示当日数据

🔧 技术架构
-----------
前端：React + TypeScript + Leaflet地图
后端：Node.js + Express + 飞书API
部署：Netlify + Render
存储：GitHub仓库

📞 技术支持
-----------
如遇问题请检查：
1. 网络连接是否正常
2. 环境变量是否正确配置
3. 飞书数据格式是否符合要求
