印尼雅加达冰柜配送地图系统 - 本地测试指南
=========================================

🚀 快速启动
----------
1. 双击 "本地启动系统.sh" 文件
2. 系统会自动：
   - 检查和安装依赖
   - 启动前后端服务
   - 打开浏览器访问系统

📍 访问地址
----------
- 前端界面: http://localhost:3000
- 后端API: http://localhost:5000
- 健康检查: http://localhost:5000/health

🔑 登录信息
----------
用户名: One Meter
密码: prioritaspelayanan

📊 测试数据
----------
系统包含12个示例订单，分布在雅加达不同区域：
- PO Freezer订单（绿色标记）: 8个
- Return Freezer订单（红色标记）: 4个
- 已完成订单（灰色标记）: 3个

🗺️ 系统功能测试
--------------
1. 地图标记显示
   - 总部位置（金色星形标记）
   - 订单位置（彩色圆形标记）
   - 点击标记查看详细信息

2. 地图操作
   - 缩放和拖拽
   - 切换标准地图/卫星视图
   - 获取当前位置

3. 统计面板
   - 总订单数量显示
   - 不同类型订单计数
   - 完成状态统计

4. 数据刷新
   - 点击刷新按钮（🔄）
   - 模拟数据同步过程

⚠️ 注意事项
-----------
1. 本地测试模式使用模拟数据
2. 刷新功能在本地模式下会显示测试提示
3. 确保端口3000和5000未被占用
4. 需要稳定的网络连接加载地图瓦片

🔧 故障排除
-----------
如果系统无法启动：
1. 检查Node.js是否正确安装
2. 确认端口没有被其他程序占用
3. 查看 logs/ 目录下的日志文件
4. 重新运行启动脚本

📝 日志文件
----------
- logs/frontend.log - 前端服务日志
- logs/backend.log - 后端服务日志

⏹️ 停止系统
-----------
- 关闭终端窗口
- 或在终端中按 Ctrl+C

💡 开发建议
----------
1. 本地测试通过后再配置真实的飞书API
2. 可以修改 public/markers.csv 文件测试不同数据
3. 检查浏览器控制台获取详细错误信息
